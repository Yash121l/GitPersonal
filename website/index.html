Project documentation

Forge is a self-hosted Git learning project with a deployable backend and full implementation notes.

This site documents how the current codebase works: authentication, repository storage, Git transport over HTTP and SSH, background maintenance, local deployment, and the data model behind it. The goal is to make the system understandable and easy to self-host.

Project summary

  • Author: Yash Lunawat, India
  • Purpose: learning how GitHub-style systems work by building one
  • Current scope: backend, transport, storage, deployment, and docs
  • Deployment model: self-hosted on your own machine or server
  • Status: foundation implemented, product surface still incomplete

About the project

This documentation is meant to be practical.

Forge is not documented as a marketing site. It is documented as an engineering project. The pages here explain what is running today, how it is structured, and which parts are still missing before the system becomes a broader Git hosting product.

Self-hosting is the default posture for the project. The website, deployment flow, and documentation are written so the code can be studied and run without relying on external managed platforms.

Current implementation

Working backend foundation

The current codebase includes smart HTTP, SSH Git transport, persisted sessions, repository provisioning, and background maintenance scheduling.

Documentation coverage

Project manual, not a brochure

What is next

Still a partial product

The next layers are repository browsing, finer-grained permissions, organizations, collaboration features, and the rest of the product UI.

What is implemented today

Implemented now

  • Go service with structured logging, graceful shutdown, config validation, and health endpoints
  • JWT cookie sessions with persisted session records and logout revocation
  • Repository provisioning on disk with sharded layout and atomic staging
  • Smart HTTP transport through git-http-backend
  • SSH transport with stored public keys and git-upload-pack / git-receive-pack
  • Background repository maintenance for size/index/commit-graph refresh
  • Local runtime verified through docker-compose

Open work

Not implemented yet

  • Web UI and documentation rendering inside the product itself
  • Pull requests, issues, organizations, and collaboration permissions
  • Rate limiting, CSRF, audit trails, richer authorization, and admin workflows
  • Webhook workers, CI runners, release flows, LFS, and search
