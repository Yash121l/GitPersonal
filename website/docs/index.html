Project Overview

Forge documentation is organized as a technical reference for the current codebase.

This documentation set describes the Forge backend as it exists today: storage, authentication, Git transport, maintenance, deployment, and testing. It separates implemented behavior from planned work.

Purpose

Forge is a self-hostable Git platform built as a learning exercise. The purpose of this site is to document the implementation in a way that is useful for reading, testing, and running the project.

The project is written to run on infrastructure you control. The documentation reflects that by focusing on local deployment, explicit operational boundaries, and real transport behavior.

Current Status

Implemented
  • HTTP API foundation
  • PostgreSQL-backed metadata store
  • JWT + persisted sessions with revocation
  • Repository provisioning and deletion
  • Smart HTTP Git transport
  • SSH Git transport
  • Background maintenance/indexing
  • Local `docker-compose` runtime verification
Not Implemented Yet
  • Web UI inside Forge itself
  • PRs, issues, reviews, teams, orgs
  • Fine-grained permissions beyond owner/public rules
  • Webhooks, CI pipelines, releases, LFS
  • Admin UX, metrics dashboards, rate limiting

Guiding Principles

  • Self-hosting first: the stack must be runnable locally and on internal servers.
  • Honest scope: implemented features are documented separately from future work.
  • Boring infrastructure: Git repos on disk, PostgreSQL for metadata, explicit startup checks.
  • Learning value: modules are organized so readers can understand system boundaries.
  • Testable transport: Git-over-HTTP and Git-over-SSH are verified in automated tests.

Glossary

Bare repository
The server-side Git repository format used for transport and shared access. Forge stores repositories this way on disk.
Smart HTTP
The Git-aware HTTP protocol served through `git-http-backend` for clone, fetch, and push.
Maintenance scheduler
The background worker that updates repository size metadata and runs `git gc --auto` and commit-graph refresh.
Session revocation
The act of invalidating a JWT by persisting its `jti` in the database and marking it revoked on logout.
Self-hosting
Running the platform on infrastructure you control, with direct access to the code, configuration, and data.
