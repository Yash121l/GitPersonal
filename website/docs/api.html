Interface Reference

Forge currently exposes JSON endpoints plus real Git transport over HTTP and SSH.

This page covers the full current runtime surface from the codebase: health endpoints, auth flows, repository APIs, SSH key registration, smart HTTP Git, SSH Git, request middleware, and error behavior.

Surface Area

Interface Entry point Current role
Health /healthz, /readyz Liveness and readiness checks for process, store, and repository storage.
JSON API /api/v1/* Auth lifecycle, current user lookup, SSH key registration, and repository CRUD.
Git smart HTTP /git/{owner}/{repo}.git Clone, fetch, push, and ref advertisement through git-http-backend.
Git over SSH ssh://git@host:2222/{owner}/{repo}.git Clone, fetch, and push through git-upload-pack and git-receive-pack.

Common Conventions

  • JSON endpoints return application/json.
  • JSON decoding is strict: unknown fields are rejected.
  • JSON request bodies must contain exactly one object.
  • Request IDs are returned in the X-Request-Id response header.
  • When the client sends X-Request-Id, Forge reuses it instead of generating a new one.
  • Request bodies are wrapped with an HTTP max-bytes reader using FORGE_MAX_REQUEST_BODY_BYTES. The default is 1 MiB.
  • Request contexts are bounded by FORGE_REQUEST_TIMEOUT.

The current handlers do not require a specific Content-Type header, but JSON clients should still send Content-Type: application/json.

Authentication

HTTP auth modes

  • Session cookie: browser-oriented auth using a signed JWT stored in an HTTP-only cookie.
  • HTTP Basic auth: username and password auth for Git clients and scripts.

Resolution order

Forge first attempts session-cookie authentication. If that fails, it attempts HTTP Basic auth. If both fail, the request is unauthorized.

Session behavior

  • Passwords are hashed with bcrypt using Go's default cost.
  • Each successful login or registration creates a JWT plus a persisted session row keyed by token ID.
  • JWT claims include user_id, username, role, issued-at, expiry, subject, and a token ID.
  • Logout revokes the persisted session and expires the cookie on the client.
  • The cookie is HttpOnly, SameSite=Lax, scoped to /, and marked Secure when FORGE_BASE_URL uses HTTPS.

SSH auth

  • SSH auth is public-key only.
  • Registered keys are identified by SHA-256 fingerprint.
  • Successful SSH auth updates last_used_at for the matching key.

Validation Rules

Field Rule
username Required for register/login. Registration accepts 3 to 39 characters matching ^[a-zA-Z0-9._-]+$.
password Registration requires at least 12 characters. Login checks the bcrypt hash for the stored user.
repository.name Required on create. Length 1 to 100 characters matching ^[a-zA-Z0-9._-]+$.
repository.visibility Must be public or private. Empty input is normalized to private.
repository.default_branch Optional. Empty input defaults to main.
ssh key name Required and trimmed.
ssh key public_key Required and must parse as an OpenSSH authorized key line.

HTTP Route Reference

GET /healthz

Liveness check

No authentication required. Returns process liveness only.

{
  "status": "ok",
  "name": "forge"
}

Status codes: 200

GET /readyz

Readiness check

Checks the configured store and the repository storage path.

{
  "status": "ready",
  "name": "forge"
}

Status codes: 200, 503

Failure cases: database not ready, repository storage not ready.

POST /api/v1/auth/register

Create user and start session

{
  "username": "yash",
  "password": "correct horse battery staple"
}

Creates a user with role member, hashes the password, persists a session, and sets the session cookie.

Status codes: 201, 400, 409, 500

Common 400 cases: invalid username, password shorter than 12 characters, malformed JSON.

POST /api/v1/auth/login

Authenticate existing user

{
  "username": "yash",
  "password": "correct horse battery staple"
}

Looks up the user, verifies the bcrypt hash, creates a new session row, and sets the cookie.

Status codes: 200, 400, 401, 500

Common 401 case: invalid credentials.

POST /api/v1/auth/logout

Revoke current session

If a valid session cookie is present, Forge revokes the persisted session row and expires the cookie.

{
  "status": "logged_out"
}

Status codes: 200

GET /api/v1/me

Return current user

Requires a valid session cookie or Basic auth.

{
  "user": {
    "id": 1,
    "username": "yash",
    "role": "member",
    "created_at": "2026-05-30T00:00:00Z"
  }
}

Status codes: 200, 401

POST /api/v1/keys

Register SSH public key

{
  "name": "laptop",
  "public_key": "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAA..."
}

Parses the key, computes the SHA-256 fingerprint, and stores it for the authenticated user.

Status codes: 201, 400, 401, 409, 500

GET /api/v1/repos

List repositories for the current user

Returns repositories owned by the authenticated user. It does not list public repositories or repos owned by other users.

{
  "repositories": [
    {
      "id": 1,
      "owner": "yash",
      "name": "forge",
      "description": "student git host",
      "visibility": "private",
      "default_branch": "main",
      "archived": false,
      "size_bytes": 0,
      "last_indexed_at": "2026-05-30T00:00:00Z",
      "last_maintained_at": "2026-05-30T00:00:00Z",
      "created_at": "2026-05-30T00:00:00Z",
      "updated_at": "2026-05-30T00:00:00Z"
    }
  ]
}

Status codes: 200, 401, 500

POST /api/v1/repos

Create repository

{
  "name": "forge",
  "description": "student git host",
  "visibility": "private",
  "default_branch": "main"
}

The repository service provisions the bare repository first, persists metadata second, and enqueues maintenance after a successful create.

Status codes: 201, 400, 401, 409, 500

Common 400 cases: invalid repository name, invalid visibility, malformed JSON.

DELETE /api/v1/repos/{owner}/{repo}

Delete repository

Only the repository owner or a user with role owner can delete a repository. Deletion is staged through the repository service so the filesystem repo is moved to trash before cleanup.

Status codes: 204, 401, 403, 404, 500

Git Smart HTTP

Repository URL shape

http://host:3000/git/{owner}/{repo}.git

Recognized request forms

  • /git/{owner}/{repo}.git/info/refs?service=git-upload-pack
  • /git/{owner}/{repo}.git/git-upload-pack
  • /git/{owner}/{repo}.git/info/refs?service=git-receive-pack
  • /git/{owner}/{repo}.git/git-receive-pack

Path rules

  • The URL must start with /git/.
  • The repository path must contain exactly {owner}/{repo}.git before any Git suffix.
  • If the repository segment does not end in .git, the request is rejected as not found.

Authorization rules

  • Public repositories are readable without authentication.
  • Private repositories require authentication for reads.
  • All writes require authentication and current write permission.
  • Unauthorized Git requests return 401 with WWW-Authenticate: Basic realm="Forge Git".

Write detection

Forge treats a request as write-oriented when the service query parameter is git-receive-pack or the request path ends in /git-receive-pack.

Backend execution

After auth and repository lookup, Forge invokes git-http-backend with CGI-style environment variables including GIT_PROJECT_ROOT, PATH_INFO, QUERY_STRING, REQUEST_METHOD, CONTENT_TYPE, CONTENT_LENGTH, and optionally REMOTE_USER plus AUTH_TYPE=Basic.

Status codes: 200, 401, 404, 502

Post-write behavior: successful receive-pack requests enqueue repository maintenance.

SSH Transport

Remote form

ssh://git@host:2222/{owner}/{repo}.git

Accepted usernames

  • The configured shared SSH username, default git.
  • The actual username of the authenticated key owner.

Supported commands

  • git-upload-pack owner/repo.git for clone and fetch.
  • git-receive-pack owner/repo.git for push.

Protocol limits

  • Only the SSH session channel type is accepted.
  • Only exec requests are accepted on that channel.
  • Forge does not provide a shell or interactive terminal.
  • The command parser requires exactly two fields: the Git command and the repository path.
  • The repository path must be exactly owner/repo.git.

Host key behavior

The SSH server loads the host key from FORGE_SSH_HOST_KEY_PATH. If the file does not exist, Forge generates an Ed25519 key, writes it to disk with mode 0600, and reuses it on later boots.

Middleware And Headers

Layer Current behavior
Request ID Sets X-Request-Id and exposes it in JSON error bodies when available.
Request logging Logs method, path, status, duration, remote address, and request ID through structured logging.
Panic recovery Returns 500 and logs the stack trace when a handler panics.
Security headers Sets Referrer-Policy, X-Content-Type-Options, X-Frame-Options, and a restrictive Content-Security-Policy.
Body limit Wraps the request body with http.MaxBytesReader.
Timeout Applies a request-scoped timeout using FORGE_REQUEST_TIMEOUT.

Error Model

JSON errors use a stable envelope:

{
  "error": "authentication required",
  "request_id": "optional-request-id"
}

Message text is handler-specific. For example, repository deletion can return cannot delete repository owned by another user, while unauthorized API requests return authentication required.

End-To-End Examples

curl -sS -c cookies.txt \
  -H 'Content-Type: application/json' \
  -d '{"username":"yash","password":"correct horse battery staple"}' \
  http://127.0.0.1:3000/api/v1/auth/register

curl -sS -b cookies.txt http://127.0.0.1:3000/api/v1/me

curl -sS -b cookies.txt \
  -H 'Content-Type: application/json' \
  -d '{"name":"forge","description":"student git host","visibility":"private","default_branch":"main"}' \
  http://127.0.0.1:3000/api/v1/repos

curl -sS -b cookies.txt \
  -H 'Content-Type: application/json' \
  -d '{"name":"laptop","public_key":"ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAA..."}' \
  http://127.0.0.1:3000/api/v1/keys

git remote add origin http://yash:correct%20horse%20battery%20staple@127.0.0.1:3000/git/yash/forge.git
git push origin HEAD:refs/heads/main

GIT_SSH_COMMAND='ssh -i ~/.ssh/id_ed25519 -o IdentitiesOnly=yes -o StrictHostKeyChecking=no' \
  git ls-remote ssh://git@127.0.0.1:2222/yash/forge.git
